# Sticky Business — русская локализация

Инструменты для извлечения и применения русской локализации к JSON-контейнерам игры Sticky Business (формат Unity).

## Структура проекта

| Папка / файл | Описание |
|--------------|----------|
| `containers/` | Исходные JSON-файлы локализации из игры |
| `extracted/` | Текстовые файлы с парами «ключ: значение» (ключ и русский перевод) |
| `containers_ru/` | Результат: JSON с подставленной русской локализацией |
| `extract_loc_keys.py` | Извлекает ключи и английские значения из контейнеров в `extracted/` |
| `apply_russian_loc.py` | Копирует контейнеры в `containers_ru/` и подставляет переводы из `extracted/` |

## Как пользоваться

### 1. Извлечь ключи из контейнеров

Из каждого JSON в `containers/` достаются пары **ключ : английское значение** и сохраняются в `extracted/` в виде `.txt` с тем же именем:

```bash
py -3 extract_loc_keys.py
```

После этого в `extracted/` можно править строки: заменить английский текст на русский, сохраняя формат `ключ: русский текст`.

### 2. Применить русскую локализацию

Скрипт копирует все файлы из `containers/` в `containers_ru/`, подставляет русские значения из соответствующих файлов в `extracted/` и обрабатывает особый случай с языком в меню:

```bash
py -3 apply_russian_loc.py
```

- Для каждой записи в JSON подставляется русский текст в **первый язык** (тот же слот, откуда читается английский в `extract_loc_keys.py`).
- В файле **Menus** для ключа `LANGUAGE_ENGLISH` **все** языковые значения (не только английское) заменяются на `Русский`.

Готовые файлы для подмены в игре лежат в **`containers_ru/`**.

## Требования

- Python 3.10+ (используются аннотации вида `str | None`)
- Стандартная библиотека (json, pathlib, shutil, os)

## Формат extracted

В каждом `.txt` в `extracted/` — по одной строке на запись:

```
KEY_NAME: Текст на русском
ANOTHER_KEY: Другой текст, можно с переносами заменять на пробелы
```

Разделитель — первое вхождение `": "` (двоеточие и пробел), поэтому в самом тексте перевода лучше не использовать такую комбинацию в начале строки.
